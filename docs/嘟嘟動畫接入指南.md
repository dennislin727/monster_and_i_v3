# 嘟嘟（`dudu`）動畫接入指南

當各動畫畫好後，依本指南把圖接回專案。**不必改程式**，只要在 Godot 裡換 `SpriteFrames` 與（可選）`PetResource` 的圖示。

---

## 1. 你會動到的檔案

| 用途 | 路徑 |
|------|------|
| **動畫軌與幀（主要工作）** | `res://resources身分證/pet/dudu_sprite_frames.tres` |
| **寵物身分／技能／指向動畫** | `res://resources身分證/pet/dudu_pet.tres` |
| **目前占位圖（可刪或留）** | `res://assets圖片_字體_音效/寵物/嘟嘟/placeholder_empty.png` |

建議把正式圖放在同一資料夾下，例如：

`res://assets圖片_字體_音效/寵物/嘟嘟/idle_down/`、`run_side/`、`spell/`…（資料夾名僅建議，可自訂。）

---

## 2. 動畫名稱必須對齊（勿改名）

出戰寵物邏輯（`PetCompanion`）與史萊姆同一套命名；**下列名稱請在 `SpriteFrames` 裡保留**（可多幀，不要改字串）：

- **待機／跑步（含方向）**  
  `idle_down`、`idle_side`、`idle_side_down`、`idle_side_up`、`idle_up`  
  `run_down`、`run_side`、`run_side_down`、`run_side_up`、`run_up`
- **戰鬥／演出**  
  `attack_down`、`attack_side`、`attack_side_down`、`attack_side_up`、`attack_up`  
  `hit_down`、`hit_side`、`hit_up`  
  `spell`（治療與翻土技能預設都會用到）  
  `happy`（召喚慶祝等）  
  `die`

若某條暫時沒圖：可仍用 1 幀占位，或複製最相近方向的一條暫代，避免執行期 `play` 找不到動畫名。

---

## 3. 在 Godot 編輯器裡接入（建議流程）

1. **匯入 PNG**  
   把畫好的序列圖放进專案（建議上述 `寵物/嘟嘟/…` 下）。Godot 會自動產生 `.import`，無需手刻。

2. **開啟 `dudu_sprite_frames.tres`**  
   - 在 **檔案系統** 雙擊 `resources身分證/pet/dudu_sprite_frames.tres`。  
   - 下方會列出每個 **Animation**（應已與上表同名）。

3. **逐條替換幀**  
   - 點選某動畫（例如 `idle_down`）。  
   - 刪掉占位幀或 **Add frames from sprite sheet / 從檔案加入**，依你習慣選「單張連續」或「Sprite Sheet」。  
   - 調整 **FPS / Speed**、是否 **Loop**（待機／跑步通常 `loop`；攻擊、`spell`、`die`、`happy` 多為單次）。

4. **檢查 `spell` 長度**  
   治療觸發延遲仍參考 `skill_slime_heal.tres` 的 `trigger_delay`；若你把 `spell` 變長／變短，之後可改技能 Resource 的時序欄位對齊手感。

5. **（可選）換清單頭像**  
   開啟 `dudu_pet.tres`，把 **Icon** 改成單張代表圖（例如待機正面），與 `sprite_frames` 分開沒關係。

6. **存檔**  
   `Ctrl+S` 存 `SpriteFrames` 與 `dudu_pet.tres`。

---

## 4. 遊戲內快速驗收

- **寵物頁**：能選到嘟嘟（若已入庫）、頭像與名稱正確。  
- **出戰**：待機／跟隨跑步是否撥對應軌；召喚時是否有 `happy`（或後備動畫）。  
- **治療**：主角受傷時是否撥 `spell`，補血節奏是否合理。  
- **家園**：進家園區後是否在土格旁撥 `spell`（翻土技能沿用同一動畫名時）。

若出現 `There is no animation with name ''` 或找不到某動畫，優先檢查該動畫名是否**完全一致**（含大小寫）。

---

## 5. 與「怪物轉化」的關係（備忘）

嘟嘟是**獨立** `PetResource`，預設**不**經史萊姆封印轉化。  
之後若要做「某劇情送嘟嘟」，在劇情／NPC／除錯流程裡 **`duplicate(true)` 一份 `dudu_pet.tres` 再交給 `PetManager` 入庫**即可（與既有封印入庫分開實作）。

---

## 6. 留給未來的你（檢查清單）

- [ ] 所有必備動畫名在 `dudu_sprite_frames.tres` 內都存在且至少 1 幀以上  
- [ ] `idle_*` / `run_*` 迴圈、`spell` / `attack_*` / `die` 節奏順眼  
- [ ] `dudu_pet.tres` 的 **Icon** 已換成正式小圖（可選）  
- [ ] 實機：出戰、補血、家園翻土各試一次  

完成後若要我幫你對技能 `trigger_delay` 或缺動畫時的後備邏輯微調，把 `dudu_sprite_frames.tres` 與你希望的手感一句話寫在對話裡即可。

# 復原對照（索引）

本目錄保存「對話串收斂後的最終定版」，供專案遺失或快照不完整時**手動疊版**對照。架構規範仍以專案根目錄的 [`ARCHITECTURE.md`](../../ARCHITECTURE.md) 為準。

---

| 檔案 | 內容重點 |
|------|----------|
| [`復原對照_兩串結果整合.md`](復原對照_兩串結果整合.md) | **功能與邏輯**：PetUI 改名流程、`SignalBus`／`PetManager`、場上名稱同步、左清單兩行樣式；頭飾錨點、翻轉、`monster_resource`、動畫回退、怪物 `die` 幀等。 |

兩份文件在 **PetUI 左欄清單**（兩行、` Lv%d` meta）有重疊；以「兩串結果整合」為**行為與程式約定**，「UI 第二串」為**視覺定版**；若衝突，先滿足 `ARCHITECTURE.md` 與可執行 smoke test。

## 建議疊版順序（總體）

1. 以可進主場景、少紅字的基底為主。
2. **先**依 [`復原對照_兩串結果整合.md`](復原對照_兩串結果整合.md) 內「若找不到完整快照：建議疊版順序」：第一串（改名／清單／訊號／`PetCompanion`）→ 第二串（頭飾錨點與動畫）。
3. **再**依 [`復原對照_UI第二串整理.md`](復原對照_UI第二串整理.md) 第五節「次要疊版建議順序」補 UI（字色 → Inventory 說明區 → PetUI 右欄／左欄 → 頁籤 → 封印／瞬移）。

每完成一組請跑兩份文件中的驗收清單（「快速驗收」與「UI 第二串」可併行勾選）。

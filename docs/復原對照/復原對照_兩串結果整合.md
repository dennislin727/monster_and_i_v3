## 第一串（UI / 改名 / 清單版面）

### A. PetUI 改名流程

- `scenes場景/ui介面/PetUI.gd`
  - 右側名稱區有「改名」入口。
  - 可點按鈕觸發改名輸入框。
  - 可點等級/綽號列觸發改名輸入框（對應 `gui_input`）。
  - 輸入限制長度 12。
  - 提交只發 `SignalBus.pet_nickname_change_requested(...)`。
  - 改名彈窗輸入框為「純空白進場」：
    - 不預填名稱。
    - 不顯示 placeholder / 提示文。
    - 背景透明（移除輸入框背景）。

- `scenes場景/ui介面/PetUI.tscn`
  - 世界名稱顯示字級有調小（曾從 12 改到 10）。
  - 中途有做過 `RowRename`，但後續有退回後再重排，最終以「需求結果」為準（非中途結構）。

### B. SignalBus / PetManager 改名協議

- `src腳本/autoload管理員/SignalBus.gd`
  - `pet_nickname_change_requested(pet_data, nickname)`
  - `pet_nickname_changed(pet_data)`

- `src腳本/autoload管理員/PetManager.gd`
  - 監聽改名請求，驗證後寫入 `pet_data.nickname`。
  - 空字串 fallback：`pet_name` / `pet_id`。
  - 改名後廣播：
    - `pet_nickname_changed`
    - `pet_roster_changed`
    - `pet_active_changed`（該寵為 active 時）

### C. 場上寵物名稱同步

- `src腳本/entities/pets/PetCompanion.gd`
  - 有頭上 `NameLabel`。
  - 監聽 `pet_nickname_changed`，同一隻寵物即時更新。
  - 召喚時會帶入目前綽號。

- `scenes場景/entities主角_怪物_寵物/寵物/PetCompanion.tscn`
  - 有對應名稱顯示節點設定。

### D. 左側寵物清單最終樣式

- `scenes場景/ui介面/PetUI.gd`
  - 左側清單改為兩行：
    - 第一行：寵物名稱
    - 第二行：等級 + 出戰標記
  - 第二行 meta 前置 1 空格（最終版本）：
    - `var meta := " Lv%d" % maxi(1, p.level)`

## 第二串（主角/怪物/寵物 頭飾錨點與動畫）

### A. 主角頭飾錨點防呆與翻轉

- `src腳本/entities/PlayerController.gd`
  - 命中 `frame_offsets` 但 offset 為 `Vector2.ZERO` 時：
    - 會沿用同動畫「前一幀最近非零 offset」。
  - 有修成強型別版本（避免 Variant 推斷警告當錯誤）。
  - `flip_h == true` 時，錨點 `x` 會鏡像（`x = -x`）。

### B. 怪物/寵物翻轉規則同步

- `src腳本/entities/monsters/MonsterBase.gd`
- `src腳本/entities/pets/PetCompanion.gd`
  - 錨點更新簽名有納入 `flip_h`（避免同幀翻面不刷新）。
  - 翻面時錨點 `x` 鏡像。
  - 不影響主角既有手調座標資料。

### C. 編輯器預覽與資源工具化（怪物/寵物）

- `resources身分證/monster/monster_resource.gd`
  - 補齊 offset 列工具（含 add row 流程）。
  - 動畫 key 候選支援方向後綴回退。

- `src腳本/entities/monsters/MonsterBase.gd`
  - 補 editor 下即時更新鏈路（含穩定性防呆）。
  - 怪物節點可見三層調整結構（基準/動畫/幀）。
  - 針對 placeholder / editor 生命週期問題做防護。

- `src腳本/entities/pets/PetCompanion.gd`
  - editor 預覽刷新行為對齊主角/怪物。

### D. 動畫名後綴回退規則

- `resources身分證/monster/monster_resource.gd`
- `src腳本/entities/pets/PetCompanion.gd`
- `src腳本/entities/monsters/MonsterBase.gd`
  - 方向後綴找不到時回退主名：
    - `attack_side` -> `attack`
    - `attack_up` -> `attack`
    - `hit_down` / `hit_side` / `hit_up` -> `hit`

### E. 寵物與怪物場景對齊（頭飾偏移）

- `scenes場景/entities主角_怪物_寵物/寵物/PetCompanion.tscn`
  - 曾對齊怪物場景：
    - `AnimatedSprite2D` 位置調整（去除 `(6, -1)` 偏移）
    - `AccessoryPoint` 對齊怪物基準

### F. 死亡動畫 die 幀錨點在 runtime 生效

- `src腳本/entities/monsters/MonsterBase.gd`
  - `is_dead` 分支不再直接跳出。
  - 會在 return 前持續：
    - `_update_accessory_anchor()`
    - `_update_headwear_visual()`
  - 目標：die 的 `frame_offsets` 在實機運行時仍逐幀生效。

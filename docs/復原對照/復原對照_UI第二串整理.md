## 二、最終定版（可視為「UI 第二串終點」）

### A. InventoryUI

- 頁籤順序：
  - `頭飾` 在左、`道具` 在右
  - 進入背包預設顯示 `頭飾`
- 下方說明區：
  - 需為「真正有底色」的區塊（Panel 化，不是只有文字）
  - 空背包時不要出現灰色假卡片占位
- 按鈕字色：
  - `背包/寵物`（開關）按住、按下、hover、focus 都維持棕色，不轉白

### B. PetUI（右欄資訊）

- 無寵物狀態（`active_pet == null`）：
  - 右側資訊欄與分隔線可隱藏（整欄隱藏）
  - 不顯示出戲文字/顏色骨架
- 改名引導文案：
  - 內部識別 ID 不作為玩家主顯示
  - 顯示「（綽號）可改名」這類玩家導向文案
- 技能資訊字級一致：
  - `故事 / 技能 / 冷卻` 字級與「技能名 Lv1」一致（小字級）
- 放生確認文案：
  - 寵物名稱不加粗（正常字重）

### C. PetUI（左欄清單）

- 清單列為兩行資訊：
  - 第一行：名稱
  - 第二行：等級 + 出戰標記
- 第二行 meta 最終為：
  - `var meta := " Lv%d" % maxi(1, p.level)`（前 1 空格）
- 列間虛線：
  - 最終拿掉（避免間距過大）
- 選取反饋：
  - 保留「選到該列有較深底色」提示
  - 選取底色有圓角
- 行高/間距：
  - 標題列高度與清單列同高（最終同步）
  - 間距統一為 8

### D. 全局按鈕視覺規則（此串定版）

- `出戰/休息/坐騎/下騎/放生`：
  - 所有狀態字色固定棕色（含 pressed/disabled）
  - disabled 不得造成按鈕尺寸跳動
  - 放生按鈕靠右，與主要操作區隔
- `封印/瞬移`：
  - 深色底（接近道具說明區色系）
  - hover 以透明度變化提示
  - 字色固定棕色，不轉白
